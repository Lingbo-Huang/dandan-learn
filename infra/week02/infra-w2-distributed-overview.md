## 一、为什么需要分布式训练？

### 1.1 单卡的天花板

```
GPT-3 (175B 参数) 的显存需求估算：
  参数本身 (FP16):  175B × 2 bytes = 350 GB
  梯度 (FP16):      175B × 2 bytes = 350 GB
  优化器状态 (FP32): 175B × 8 bytes = 1400 GB (Adam: m + v + 主权重)
  ─────────────────────────────────────────────
  总计:  ≈ 2100 GB

  A100 80GB 的显存：80 GB
  需要多少张 A100？ 约 26 张（仅参数+梯度+优化器，不含激活值）
```

**核心矛盾**：模型越来越大，单张 GPU 装不下了。

### 1.2 三大瓶颈

```
┌────────────────────────────────────────────────────────┐
│  分布式训练的三大瓶颈                                    │
│                                                          │
│  ① 显存瓶颈 ─── 模型 + 梯度 + 优化器 > 单卡显存         │
│     解法：模型并行、ZeRO                                 │
│                                                          │
│  ② 计算瓶颈 ─── 数据量大，训练太慢                      │
│     解法：数据并行（多卡同时处理不同 batch）              │
│                                                          │
│  ③ 通信瓶颈 ─── GPU 间通信拖累整体速度                  │
│     解法：梯度压缩、重叠通信与计算、拓扑感知调度          │
└────────────────────────────────────────────────────────┘
```

## 二、分布式训练的三大范式

### 2.1 全景对比图

```
                    分布式训练范式
                         │
         ┌───────────────┼───────────────┐
         │               │               │
    数据并行          模型并行         混合并行
  Data Parallel    Model Parallel   Hybrid Parallel
         │               │
    ┌────┴────┐     ┌────┴────┐
    │         │     │         │
   DP        DDP  张量并行  流水线并行
 (旧)       (新) Tensor-P  Pipeline-P
```

### 2.2 核心区别一句话

| 范式 | 核心思路 | 适用场景 |
|------|---------|---------|
| **数据并行** | 每张卡有完整模型副本，各自处理不同数据，梯度汇总 | 模型能放进单卡 |
| **张量并行** | 将矩阵运算拆开，不同卡算矩阵的不同部分 | 超大模型，单层放不进单卡 |
| **流水线并行** | 将模型按层切割，不同卡负责不同层 | 超深模型 |
| **ZeRO** | 数据并行的变体，把状态分摊到各卡（不是复制） | 数据并行 + 显存极度紧张 |

## 三、并行维度可视化

### 3.1 数据并行

```
Batch = [s1, s2, s3, s4, s5, s6, s7, s8]

GPU 0: [s1, s2] → Model Copy 0 → grad0 ─┐
GPU 1: [s3, s4] → Model Copy 1 → grad1 ─┤ All-Reduce → 平均梯度 → 更新参数
GPU 2: [s5, s6] → Model Copy 2 → grad2 ─┤
GPU 3: [s7, s8] → Model Copy 3 → grad3 ─┘
```

**显存代价**：每张卡都存一份完整参数 + 梯度 + 优化器状态。

### 3.2 流水线并行

```
模型分4层组，4张卡流水线执行：

        Micro-batch:  mb1  mb2  mb3  mb4
GPU 0 (Layer 1-3):   [F]  [F]  [F]  [F]  [B]  [B]  [B]  [B]
GPU 1 (Layer 4-6):        [F]  [F]  [F]  [F]  [B]  [B]  [B]  [B]
GPU 2 (Layer 7-9):              [F]  [F]  [F]  [F]  [B]  [B]  [B]  [B]
GPU 3 (Layer 10-12):                  [F]  [F]  [F]  [F]  [B]  [B]  [B]

F=前向  B=反向  气泡（idle）存在于流水线边界
```

### 3.3 张量并行

```
矩阵乘法 Y = XW，W 按列切分：

GPU 0: Y0 = X · W[:, :N/2]    ←─┐
GPU 1: Y1 = X · W[:, N/2:]    ←─┤ All-Gather → Y = [Y0, Y1]
                                  │
         X 被广播到所有 GPU ───────┘
```

## 四、通信原语：All-Reduce、All-Gather、Reduce-Scatter

### 4.1 图解

```
All-Reduce (数据并行梯度汇总):
  GPU0: [a0] ─┐
  GPU1: [a1] ─┤ → 每张 GPU 都得到 sum/avg([a0,a1,a2,a3])
  GPU2: [a2] ─┤
  GPU3: [a3] ─┘

Reduce-Scatter (ZeRO 第一步):
  每张 GPU 各持有完整梯度 → 汇总后每张 GPU 只保留 1/N 分片

All-Gather (ZeRO 第二步):
  每张 GPU 持有 1/N 参数分片 → 广播后每张 GPU 获得完整参数
```

### 4.2 通信量对比

| 通信模式 | 通信量 | 典型用途 |
|---------|-------|---------|
| All-Reduce | 2(N-1)/N × data | DDP 梯度同步 |
| Reduce-Scatter | (N-1)/N × data | ZeRO 梯度切分 |
| All-Gather | (N-1)/N × data | ZeRO 参数恢复 |
| Ring All-Reduce | = All-Reduce，带宽最优 | 实际 DDP 实现 |

## 六、如何选择并行策略？

```
决策树：

模型能放进单卡？
    ├── 是 → 用 DDP，简单有效
    └── 否 → 模型有多大？
              ├── 10B 以下 → ZeRO-3 (DeepSpeed/FSDP)
              └── 10B 以上 → 3D 混合并行 (TP + PP + DP)
                             ├── 层数深 → 增加 PP 维度
                             └── 层宽 → 增加 TP 维度
```

## 七、性能指标

| 指标 | 定义 | 工具 |
|------|------|------|
| MFU (模型浮点利用率) | 实际 FLOPS / 理论峰值 FLOPS | 手算或 pytorch profiler |
| 吞吐量 | tokens/sec 或 samples/sec | 日志统计 |
| 通信开销占比 | 通信时间 / 总时间 | NCCL profiler |
| 显存利用率 | 实际使用 / 卡总显存 | nvidia-smi |

**GPT-3 的 MFU 参考值**：约 30-50%（A100 SXM4）；做到 50%+ 是优秀水平。

## 小结

- 分布式训练的本质是**用多卡解决显存瓶颈**，同时**提升训练速度**
- 三大范式：**数据并行**（每卡完整模型）、**模型并行**（模型切分到多卡）、**ZeRO**（数据并行的显存优化变体）
- 通信原语是理解并行策略的关键：All-Reduce、Reduce-Scatter、All-Gather
- 入门先学 DDP → ZeRO → DeepSpeed，大规模场景再考虑 3D 并行
