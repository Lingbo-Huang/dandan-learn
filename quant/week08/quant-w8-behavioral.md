# D6 · 行为面试与项目展示

> 技术过关只是门票，行为面试决定你是否适合这个团队。

---

## 1. 行为面试的 STAR 方法

| 要素 | 含义 | 时间分配 |
|------|------|---------|
| Situation | 背景/情境 | 10% |
| Task | 你的任务/责任 | 15% |
| Action | 你做了什么（重点！）| 60% |
| Result | 结果与反思 | 15% |

## 2. 高频行为面试题与参考答案

**Q：描述一个你主导的量化项目**

```
框架：
S: 在 XX 实习/课题中，团队需要提升因子选股策略的稳健性
T: 我负责设计因子评价体系和防过拟合方法
A:
  1. 调研了 IC/ICIR、分层收益等评价指标，建立了标准化评估框架
  2. 引入 Walk-Forward 验证替代随机 CV，避免时序数据的未来泄露
  3. 发现原有策略存在严重的幸存者偏差，补充了历史退市股票数据
  4. 优化后在样本外测试中验证，IC 均值从 0.02 提升到 0.04
R: 策略样本外夏普比从 0.8 提升到 1.2，同时逻辑更清晰、更可解释
反思：如果重来，会更早做样本外测试，而不是在最后才发现过拟合
```

**Q：你遇到过什么困难的技术问题，如何解决？**

```
参考框架：
S: 在计算 Fama-MacBeth 横截面回归时，发现各期系数方差极大
T: 需要找出原因并提出更稳健的估计方法
A:
  1. 分析发现：部分月份小市值股票样本极小，导致估计不稳定
  2. 研究了 Newey-West 调整的截面标准误估计
  3. 对比了不同过滤条件（最低市值、最低流动性）对结果的影响
  4. 最终加入流动性过滤 + Newey-West 标准误，结果稳健
R: 因子的 t 统计量从波动在 1-4 之间稳定到 2.5 左右，更可信
```

**Q：你如何处理团队中的分歧？**

```
不要说：我说服了他们（显得强势）
不要说：最后跟着大家的意见（显得没主见）

好的回答：
"我们对因子标准化的方法有分歧（Z-score vs. 排名）。
我提议各自实现一版，用相同数据 A/B 测试 IC 差异。
结果两种方法的 IC 相近，但排名法在极端市场中更稳定。
最终团队采用了排名法，并在报告中说明了两种方法的权衡。"

要点：数据说话，尊重过程，有建设性
```

**Q：你为什么对量化感兴趣？**

```
避免套话："我对数学和金融都感兴趣"（人人都这么说）

个人化的回答：
"大学时接触了行为金融学，发现市场存在系统性偏差，
但散户无法利用它（情绪、注意力、执行力）。
量化投资通过代码消除人的情绪，系统性捕捉这些偏差，
让我觉得这是一种公平且高效的博弈方式。
加上我本身喜欢写代码，发现了一种能把技术直接转化为价值的路径。"
```

## 3. 项目展示的结构

**推荐准备 3 个项目**，不同深度：

1. **核心项目（深度展示）**：
   - 完整 Pipeline：数据 → 因子 → 回测 → 评估
   - 有样本外测试，有可运行代码
   - 准备 20 分钟深度讲解

2. **辅助项目（广度展示）**：
   - 展示你对不同方向的了解（如：ML因子/统计套利/事件驱动）
   - 5 分钟概述

3. **技术项目（工程能力）**：
   - 展示代码质量（GitHub）
   - 强调系统设计思维

## 4. 常见追问的陷阱

| 面试官说 | 潜台词 | 正确应对 |
|---------|-------|---------|
| "这个策略很好，实盘了吗？" | 质疑可实施性 | 说明容量/成本分析，或坦诚是研究阶段 |
| "IC 0.04 不高吧？" | 测试你对数字的理解 | 解释月频 IC > 0.03 是业界可用水平 |
| "这个策略别人都在用吧？" | 质疑独特性 | 说明实现细节的差异，或承认同质化风险 |
| "你了解我们公司的策略吗？" | 考察准备是否充分 | 要提前做功课！ |

## 5. 反问问题（展示真实兴趣）

- "团队目前主要做哪些方向的因子研究？"
- "策略的迭代周期是怎样的？从立项到实盘大概多久？"
- "如何评估一个研究员的工作质量？"
- "技术团队和研究团队是如何协作的？"
- "您觉得现在加入，成长最快的方向是什么？"

---

## 小结

行为面试的本质：让面试官相信你能在这个团队中有效工作。
- 技术能力：体现在项目细节
- 合作能力：体现在冲突处理
- 成长潜力：体现在反思和迭代
- 文化匹配：体现在价值观和工作方式
