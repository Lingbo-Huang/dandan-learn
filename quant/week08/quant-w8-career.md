# D5 · 岗位解析与简历建议

> 理解岗位差异，才能精准准备。量化不是一个岗位，是一个生态。

## 2. 量化研究员 vs. 策略工程师

### 量化研究员（Alpha Research）

**关注**：Alpha 的来源和有效性

技能栈：
- Python（Pandas, NumPy, Scikit-learn）
- 统计学（假设检验、时序分析）
- 金融知识（因子理论、市场微结构）
- 报告写作能力

典型工作日：
- 提出新因子假设
- 计算因子，分析 IC/ICIR
- 回测策略，分析绩效
- 写研究报告，与 PM 沟通

### 策略工程师（Strategy Engineering）

**关注**：策略如何高效稳定地在生产环境运行

技能栈：
- C++（高性能计算、底层优化）
- Python（接口层、数据处理）
- 系统设计（低延迟、高可用）
- 金融知识（行情、订单、风控）

典型工作日：
- 维护回测系统
- 将研究员的策略转化为实盘代码
- 处理行情数据、订单管理
- 优化执行延迟

## 4. 简历建议

### 结构

```
姓名 | 邮箱 | 电话 | GitHub/Blog
学历背景（GPA, 竞赛）
实习/项目经历（STAR格式）
技能（Python/C++/R/Matlab）
论文/发表（如有）
```

### 项目描述的 STAR 格式

```
差：设计了一个多因子选股策略

好：
Situation: 研究A股中频多因子策略（月度调仓）
Task: 构建能稳定产生超额收益的量化因子体系
Action:
  - 计算动量、价值、质量、低波动4类因子，做行业市值中性化
  - 用 IC/ICIR 评价体系筛选有效因子，构建等权合成因子
  - Walk-Forward 样本外验证，避免过拟合
Result:
  - 样本外年化超额收益 8%，夏普比率 1.3
  - 月度 Rank IC 均值 0.045，ICIR = 0.82
```

## 5. 面试八大误区

1. **只懂理论，不懂工程**：面试官最怕你只能说不能做
2. **代码不清洁**：命名随意，无注释，逻辑混乱
3. **忽视风险**：只说策略有多好，不说陷阱
4. **数字不精确**：说"效果不错"而不是"IC=0.04，ICIR=0.7"
5. **不了解岗位**：不知道应聘机构的策略风格
6. **夸大项目**：面试官会深挖，虚报很容易被识破
7. **不懂 A 股特殊性**：用美股理论直接套用
8. **没有经济学逻辑**：因子有效但不知道为什么
