# D4 · 量化案例面试

> 案例题考察你能否把量化方法论转化为完整的解决方案。面试官想看到系统性思维。

---

## 1. 案例题的标准答题框架

```
1. 理解问题 → 澄清假设
2. 建立框架 → 分步拆解
3. 方法设计 → 选择工具
4. 风险分析 → 陷阱识别
5. 实施计划 → 资源需求
```

---

## 2. 经典案例：设计一个动量因子

**题目**：请设计一个针对A股的动量因子，并说明如何验证其有效性。

```
标准答题框架：

一、因子定义
   - 定义：过去 N 个月（跳过最近 M 个月）的累积收益率
   - 针对A股调整：使用 6 个月回看期（而非 12 个月），跳过最近 1 个月
   - 原因：A股散户主导，长期动量弱，短期反转强

二、数据处理
   - 数据源：日度前复权收盘价
   - 处理停牌：停牌期间价格不变，需要过滤
   - 处理退市：使用历史全量股票池（包含退市股）

三、信号计算
   - 月度重采样计算累积收益
   - 截面 Winsorize（去 2.5% 极端值）
   - 市值 + 行业中性化
   - Z-score 标准化

四、有效性验证
   - IC 分析：计算月度 Rank IC，目标 |IC| > 0.03，ICIR > 0.5
   - 分层分析：5组，看 Q5-Q1 多空组合
   - 统计检验：t 检验 IC 均值是否显著不为零
   - 时间稳定性：不同子期间 IC 是否一致

五、风险分析
   - Momentum Crash 风险（市场急速反弹时亏损）
   - 行业集中度风险
   - 换手率与交易成本分析
```

## 3. 经典案例：评估一个已有策略

**题目**：给你一个策略的回测报告，夏普比率 2.5，年化收益 30%。如何评估它是否值得实盘？

```
系统评估清单：

1. 过拟合检测
   Q: 参数是怎么选的？测试了多少组参数？
   Q: 样本内外 IC 的比值是多少？（< 60% 则过拟合）
   Q: Walk-Forward 测试结果如何？

2. 数据质量
   Q: 是否有未来函数？（财务数据用报告期还是披露日期）
   Q: 是否包含退市股票？（幸存者偏差）
   Q: 股票池是历史成分还是当前成分？

3. 成本假设
   Q: 是否考虑了冲击成本？（光算佣金和印花税不够）
   Q: 换手率多高？假设的单边成本是多少？

4. 策略逻辑
   Q: 经济学逻辑是什么？（数据挖掘还是有理论依据）
   Q: 信号是否独特？还是已被广泛使用？

5. 容量分析
   Q: 策略持仓股票流动性如何？
   Q: 在多大 AUM 下这个夏普会下降到 1.0？

结论框架：
如果上述问题都能满意回答，则值得小规模实盘验证；
否则先修正问题再评估。
```

## 4. 经典案例：因子衰减分析

**题目**：你研究的因子最近 6 个月 IC 明显下降，请分析原因并给出建议。

```
分析框架：

1. 数据层面排查
   - 因子计算逻辑是否有 bug（先检查这个）
   - 数据源是否有变化（新的财务准则？）

2. 市场环境变化
   - 查看这 6 个月的市场风格（价值/成长/动量）
   - 分析这 6 个月大盘走势（牛/熊/震荡）
   - 因子在不同环境的历史 IC 对比

3. 因子失效的可能原因
   - 信号被市场广泛使用（套利压缩溢价）
   - 市场结构性变化（监管政策、投资者结构）
   - 因子本身存在周期性（动量因子在震荡市失效）

4. 建议
   短期：调低因子权重，增加稳健因子权重
   中期：研究新的补充信号，做动态权重
   长期：建立多因子监控体系，设置自动预警
```

## 5. 案例面试的加分点

- **假设明确**：主动说出假设，不让面试官猜
- **量化清晰**：用具体数字（IC > 0.03），不要说"效果很好"
- **风险意识**：每个方法都说潜在风险
- **迭代思维**：不追求完美，给出 MVP 方案 + 后续优化方向

## 常见追问与准备

| 面试官问 | 你应该答 |
|---------|---------|
| IC 多少算好？| 月频 > 0.03，ICIR > 0.5 |
| 怎么处理因子间相关？| 正交化 or 主成分，等权通常最稳健 |
| 为什么不用深度学习？| 数据少，过拟合风险高；需要严格验证 |
| 如何估算策略容量？| 持仓股票的 ADV × 参与率上限（5-10%）|
| 模型衰减了怎么办？| 分析原因，考虑动态权重或重新研究信号 |
