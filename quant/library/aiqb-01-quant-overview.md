## 量化交易到底是什么？

一句话：**用代码和数学做投资决策，并且自动执行**。

但这句话背后藏着很多东西：

```
你以为的量化交易：
  数据 → 模型 → 买卖 → 赚钱

真实的量化交易：
  脏数据清洗 → 无数失败的因子 → 严苛的风险控制 → 
  微薄的超额收益 → 高昂的IT基础设施 → 
  随时可能失效的策略 → 再次迭代
```

---

## 量化交易的全貌

### 1. 数据层

```
原始数据来源：
├── 行情数据（价格、成交量）
│   ├── 日频：AkShare, Tushare, Wind
│   ├── 分钟级：券商行情接口
│   └── Tick 级：交易所原始数据
├── 基本面数据
│   ├── 财务报表（利润表、资产负债表、现金流）
│   └── 行业数据、宏观数据
└── 另类数据（Alpha 的新战场）
    ├── 卫星图像（停车场车流量 → 推断零售业绩）
    ├── NLP（财报情绪分析、新闻影响）
    ├── 社交媒体（舆情数据）
    └── 信用卡消费数据
```

### 2. 因子层（Alpha 研究）

```
Alpha 因子的生命周期：
  发现 → 验证 → 容量估算 → 衰减 → 失效

为什么因子会失效？
├── 被太多人发现 → 套利空间消失
├── 市场结构变化（监管、交易机制）
└── 宏观环境变化（利率、经济周期）
```

### 3. 组合层

```
多因子 Alpha 合成
    ↓
风险模型（控制暴露）
├── 行业中性化
├── 市值中性化
└── 控制主要风险因子敞口
    ↓
组合优化
├── 最大化预期收益
├── 约束跟踪误差
└── 控制换手率
```

### 4. 执行层

```
信号 → 目标持仓 → 实际下单

执行的挑战：
├── 市场冲击：大单会推动价格不利方向移动
├── 时机：什么时候分批执行？
├── 算法交易：TWAP, VWAP, 参与率算法
└── 滑点管理：预期成交价 vs 实际成交价
```

## 量化交易者的技能图谱

```
量化研究员（QR）
├── 数学/统计：核心工具
├── 编程：Python, SQL, 有时C++
├── 金融知识：不需要很深，但要够用
└── 研究能力：提出假设 → 验证 → 迭代

量化开发（QD）
├── 系统设计：低延迟交易系统
├── C++/Java：性能关键路径
├── 网络/OS：Linux底层优化
└── 数据库：时序数据库（InfluxDB, kdb+）

量化策略（PM）
├── 全流程理解
├── 风险管理
├── 组合管理
└── 业务判断
```

## 不同频率的量化策略对比

| 策略频率 | 持仓时间 | Alpha 来源 | 门槛 |
|---------|---------|-----------|------|
| **低频（月/周）** | 数周-数月 | 因子溢价 | 低，适合个人/小团队 |
| **中频（日/小时）** | 数天 | 技术模式、事件驱动 | 中等 |
| **高频（分钟/秒）** | 秒-分钟 | 市场微观结构 | 极高（硬件+数据+牌照）|
| **超高频（HFT）** | 毫秒-微秒 | 做市、延迟套利 | 顶级（共址+专线+FPGA）|

## 量化的现实期望

**能做到的**：
- 年化超额收益 5-15%（相对基准，扣费后）
- 夏普比率 0.5-2.0
- 比纯主观投资更稳定的回撤控制

**不能做到的**：
- 每年 50% 收益（除非极高风险或高频）
- 永远不亏钱
- 一个因子用十年

**关键认识**：
> 量化投资的竞争对手不是普通散户，而是全球最聪明的数学家、工程师和科学家。
> 优势来自于更好的数据、更好的模型、更快的执行，或更好的风控。
