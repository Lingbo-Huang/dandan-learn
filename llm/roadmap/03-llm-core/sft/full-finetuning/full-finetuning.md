# 全量参数微调

## 1. 引言

在监督微调（SFT）的范畴内，根据训练过程中更新参数的范围，主要可以分为两大类：**全量参数微调**（Full Fine-Tuning）和**参数高效微调**（Parameter-Efficient Fine-Tuning, PEFT）。本章节将深入探讨**全量参数微调**。

**全量参数微调**，顾名思义，是指在微调阶段更新预训练模型的**所有参数**。这是最直接、最传统的微调方式，也是在大模型时代之前普遍采用的方法。

对于一个从传统软件开发转型的工程师来说，可以将全量参数微调类比为对一个已经写好的、功能庞大的软件库进行深度定制开发。你拥有这个库的全部源代码，并且可以根据你的具体需求，修改库中的每一个函数、每一个变量，使其完全符合你的新项目要求。

## 2. 全量参数微调的定义与过程

### 2.1 定义

**全量参数微调**（Full Fine-Tuning）是指在模型微调过程中，**所有可训练的参数**（包括所有的权重 `W` 和偏置 `b`）都会根据下游任务的数据和损失函数进行梯度计算和更新。

### 2.2 过程

1.  **加载预训练模型：** 从磁盘或云端加载预训练好的模型权重和配置。
2.  **准备数据：** 准备与目标任务相关的标注数据集，通常为输入-输出对。
3.  **设置训练配置：**
    *   **优化器：** 选择优化器（如AdamW）。
    *   **学习率：** 通常设置得比预训练阶段小，以避免破坏已学到的知识（如 `5e-5`, `2e-5`）。
    *   **批次大小（Batch Size）：** 根据GPU内存确定。
    *   **训练轮数（Epochs）：** 通常较少，因为数据集相对较小。
4.  **前向传播：** 将输入数据通过整个模型，计算输出。
5.  **计算损失：** 根据模型输出和真实标签计算损失（如交叉熵损失）。
6.  **反向传播：** 计算损失相对于**所有参数**的梯度。
7.  **参数更新：** 使用优化器根据梯度更新**所有参数**。
8.  **重复迭代：** 重复步骤4-7，直到模型在验证集上的性能达到满意水平或训练完成。

## 3. 全量参数微调的优缺点

### 3.1 优点

1.  **性能潜力最大：** 由于所有参数都可以调整，模型有最大的自由度去适配目标任务，理论上可以获得最佳的微调效果。
2.  **简单直接：** 概念清晰，实现简单，不需要引入额外的模型结构或复杂的训练技巧。
3.  **充分利用预训练知识：** 模型的每一部分都参与到目标任务的学习中，可以更充分地结合预训练知识和下游任务知识。

### 3.2 缺点

1.  **计算资源消耗巨大：**
    *   **内存需求高：** 需要存储所有参数的梯度，显存占用大。对于拥有数十亿甚至数千亿参数的大模型，这往往超出了单张或几张GPU的显存容量。
    *   **训练时间长：** 需要计算所有参数的梯度并进行更新，训练速度慢。
2.  **存储成本高：**
    *   **模型副本：** 为每个不同的任务或客户微调一个模型，都需要保存一个完整的模型副本。例如，一个1750亿参数的模型，即使每个参数用半精度（FP16）存储，也需要2 * 1750亿 / 8 / 1024^3 ≈ 32.5 GB的存储空间。为100个任务微调就需要数TB的存储。
3.  **部署复杂性高：**
    *   **服务管理：** 需要为每个微调模型独立部署和管理服务。
    *   **版本控制：** 模型版本和参数管理复杂。
4.  **灾难性遗忘风险：** 由于所有参数都在更新，如果SFT数据集较小或与预训练数据差异较大，模型可能更容易遗忘预训练阶段学到的通用知识。

## 4. 全量参数微调的应用场景

尽管存在上述缺点，全量参数微调在以下场景中仍有其价值：

### 4.1 计算资源充足的场景

*   **研究机构：** 拥有大量GPU/TPU集群，可以负担全量微调的计算开销。
*   **特定核心业务：** 对于公司最重要的、需要极致性能的模型服务，愿意投入大量资源进行全量微调。

### 4.2 对模型性能要求极高的场景

*   **基准测试（Benchmarking）：** 为了在公开数据集上获得最佳性能，作为技术实力的展示。
*   **科学实验：** 需要探索模型的最大潜力或进行严格的对照实验。

### 4.3 领域差异巨大的场景

*   当目标任务领域与预训练数据领域差异非常大时，可能需要更强的调整能力，全量微调可能比PEFT方法更有效。

## 5. 全量参数微调的实践细节

### 5.1 学习率设置

*   **通常较低：** 为了避免破坏预训练模型中已经学到的通用知识，SFT阶段的学习率通常比预训练阶段低一个数量级（例如，预训练用 `1e-4`，SFT用 `2e-5` 或 `5e-5`）。
*   **分层学习率（Layer-wise Learning Rate）：** 有时会为模型的不同层设置不同的学习率。例如，靠近输入的底层学习率更低，靠近输出的顶层学习率更高。这背后的直觉是，底层编码了更通用的语言特征，应谨慎调整；顶层更偏向任务特定，可以更大幅度调整。

### 5.2 优化器选择

*   **AdamW：** 目前最常用的优化器，结合了Adam的优点和权重衰减（Weight Decay）的正则化效果。
*   **学习率预热（Learning Rate Warmup）：** 在训练初期，从一个很小的学习率逐渐增加到设定值，有助于训练稳定。

### 5.3 正则化

*   **权重衰减（Weight Decay）：** 防止过拟合。
*   **梯度裁剪（Gradient Clipping）：** 防止梯度爆炸。

### 5.4 训练技巧

*   **早停（Early Stopping）：** 监控验证集性能，防止过拟合。
*   **检查点（Checkpointing）：** 定期保存模型状态，以便在训练中断后恢复或选择最佳模型。

## 6. 全量参数微调与参数高效微调（PEFT）的对比

| 特性             | 全量参数微调 (Full Fine-Tuning)            | 参数高效微调 (PEFT)                          |
| :--------------- | :----------------------------------------- | :------------------------------------------- |
| **更新参数**     | 所有参数                                   | 少部分参数或新增参数                         |
| **计算资源**     | 需求高（内存、算力）                       | 需求低                                       |
| **存储成本**     | 高（每个任务一个完整模型副本）             | 低（共享主干，仅存少量增量参数）             |
| **部署复杂度**   | 高（管理多个模型副本）                     | 低（可动态加载不同适配器）                   |
| **性能潜力**     | 理论最高                                   | 略低，但差距在缩小                           |
| **训练速度**     | 慢                                         | 快                                           |
| **适用场景**     | 资源充足、性能要求极高                     | 资源受限、多任务、快速迭代                   |

## 7. 总结

*   **全量参数微调**是更新预训练模型所有参数以适配下游任务的方法。
*   它具有**性能潜力最大**和**实现简单**的优点。
*   但其**资源消耗巨大**（内存、存储、计算）和**部署复杂**的缺点，在大模型时代变得尤为突出。
*   因此，全量参数微调主要应用于**计算资源充足**、**对性能要求极高**或**领域差异巨大**的特定场景。
*   随着大模型规模的不断增大，**参数高效微调**（PEFT）方法（如LoRA、Adapter、Prefix-Tuning等）已成为更主流的选择，以应对全量微调带来的挑战。
*   理解全量参数微调是掌握各种PEFT方法的基础，因为它代表了微调的“上限”。
