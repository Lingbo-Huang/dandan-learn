# SFT基础概念

## 1. 引言

在大语言模型（LLM）的训练和应用流程中，**监督微调**（Supervised Fine-Tuning, SFT）是一个承上启下的关键环节。在经历了海量无标签数据预训练的模型，虽然具备了强大的语言理解和生成能力，但其行为往往是通用的、非特定任务导向的。为了让模型能够更好地适应特定的应用场景（如对话、摘要、代码生成等），我们需要对其进行**微调**。

**监督微调**（SFT）是微调的一种主要形式，它利用**有标签的**（即输入-输出对）数据集，通过传统的监督学习方式来调整预训练模型的参数，使其在特定任务上表现更佳。

可以将SFT想象成一位已经拥有广博知识的大学生，为了准备专业考试（特定任务），需要在导师（有标签数据）的指导下，将已有的知识进行调整和聚焦，以更好地解决专业问题。

## 2. 为什么需要监督微调（SFT）？

### 2.1 预训练的局限性

*   **通用性 vs. 专业性：** 预训练模型通过在大规模通用语料上学习，获得了广泛的语言能力，但其输出往往是多样化的、非确定的，不一定符合特定场景的需求。
*   **格式与风格：** 预训练模型可能无法直接产生符合特定下游任务要求的输出格式（如对话的礼貌性、摘要的简洁性）。
*   **领域知识：** 对于特定领域（如医疗、法律、金融），预训练模型可能缺乏足够的专业知识。

### 2.2 指令遵循能力

*   **对齐（Alignment）：** 预训练模型可能无法很好地理解或遵循人类给出的指令。SFT可以帮助模型学习如何准确地理解和执行各种指令。
*   **有用性（Helpfulness）：** 使模型的输出更符合用户的实际需求，更有用。

## 3. 监督微调（SFT）的定义与过程

### 3.1 定义

**监督微调**（Supervised Fine-Tuning, SFT）是指在预训练模型的基础上，使用一个**带标签的**（Supervised）数据集来进一步训练模型。这个数据集通常由输入文本（如指令、问题、上下文）和对应的期望输出（如答案、回复、摘要）组成。

目标是调整模型的参数，使其在给定输入时，能够生成尽可能接近期望输出的预测结果。

### 3.2 数学表达

假设我们有一个由 `N` 个样本组成的监督数据集 `D = {(x₁, y₁), (x₂, y₂), ..., (xₙ, yₙ)}`，其中 `xᵢ` 是输入，`yᵢ` 是期望的输出。

SFT的目标是最小化模型预测输出 `f_θ(xᵢ)` 与真实标签 `yᵢ` 之间的差异，通常使用**损失函数** `L` 来衡量：

```
L_SFT = (1/N) * Σ (i=1 to N) L(f_θ(xᵢ), yᵢ)
```

其中 `θ` 是模型的参数。通过梯度下降等优化算法，更新 `θ` 以最小化 `L_SFT`。

### 3.3 典型的SFT数据格式

在大模型的SFT中，数据通常被组织成**提示-响应**（Prompt-Response）或**指令-输出**（Instruction-Output）对的形式：

```
{
  "instruction": "将以下英文翻译成中文。",
  "input": "Hello, how are you today?",
  "output": "你好，今天怎么样？"
}
```

或者更通用的对话格式：

```
[
  {"role": "user", "content": "你好，介绍一下你自己。"},
  {"role": "assistant", "content": "我是由阿里云开发的超大规模语言模型..."}
]
```

### 3.4 SFT的训练过程

1.  **准备数据：** 收集和构建高质量的、与目标任务相关的标注数据集。
2.  **加载预训练模型：** 加载在大规模语料上预训练好的模型权重。
3.  **定义训练目标：** 通常是最小化预测输出与真实标签之间的交叉熵损失。
4.  **训练：** 在标注数据集上进行训练，更新模型参数。
    *   **全量参数微调：** 更新模型的所有参数。
    *   **参数高效微调（PEFT）：** 只更新模型的一小部分参数（如LoRA），或添加少量可训练参数（如Adapter）。
5.  **验证与评估：** 在验证集上评估模型性能，防止过拟合。
6.  **部署：** 将微调好的模型部署到实际应用中。

## 4. SFT与其他训练阶段的关系

### 4.1 与预训练（Pre-Training, PT）的关系

*   **继承性：** SFT建立在预训练模型强大的语言表示能力基础之上。
*   **特化性：** SFT使得通用的预训练模型向特定任务或指令对齐。
*   **参数初始化：** SFT的起点是预训练模型的参数。

### 4.2 与强化学习（Reinforcement Learning, RL）的关系

*   **顺序性：** 在许多大模型的训练流程中（如RLHF），SFT是RL之前的必要步骤。RL需要一个相对不错的初始策略（Policy），而SFT提供了一个很好的起点。
*   **奖励模型（RM）训练：** RL通常需要一个独立的奖励模型来评估生成文本的质量。这个奖励模型有时也是在SFT数据或通过SFT模型生成的数据上训练的。
*   **互补性：** SFT主要优化与训练数据分布一致的性能，而RL可以优化更复杂、更主观的目标（如人类偏好）。

## 5. SFT的关键技术与挑战

### 5.1 数据质量与数量

*   **高质量数据：** SFT的效果高度依赖于标注数据的质量。噪声大或错误的标签会损害模型性能。
*   **数据多样性：** 数据集需要覆盖目标任务的各种场景和情况。
*   **数据量：** 虽然比预训练所需数据量小得多，但仍需要足够的数据来有效调整模型。

### 5.2 过拟合（Overfitting）

*   **问题：** SFT数据集通常比预训练数据集小很多，模型很容易在SFT阶段过拟合到训练数据上，导致泛化能力下降。
*   **缓解方法：**
    *   **正则化技术：** 如Dropout, Weight Decay。
    *   **早停（Early Stopping）：** 在验证集性能不再提升时停止训练。
    *   **数据增强：** 增加训练数据的多样性。
    *   **参数高效微调（PEFT）：** 限制模型的自由度，降低过拟合风险。

### 5.3 灾难性遗忘（Catastrophic Forgetting）

*   **问题：** 在SFT过程中，模型可能会“遗忘”在预训练阶段学到的通用知识和能力，尤其是在SFT数据与预训练数据差异较大时。
*   **缓解方法：**
    *   **多任务学习：** 在SFT时，同时在预训练数据和SFT数据上训练。
    *   **正则化：** 限制参数更新的幅度，使其不要偏离预训练参数太远（如L2 regularization on weight difference）。
    *   **知识蒸馏：** 使用预训练模型作为教师模型，引导SFT模型的训练。

### 5.4 指令微调（Instruction Tuning）

这是SFT的一个重要分支或应用形式。

*   **目标：** 让模型学会理解和遵循各种自然语言指令。
*   **数据：** 由各种任务的指令-输出对组成。
*   **效果：** 大大提升模型的通用性和交互性，使其能像一个“通用助手”一样工作。
*   **代表工作：** Flan系列（Flan-T5, Flan-PaLM）、GLM、LLaMA2-Chat等。

## 6. SFT的应用实例

### 6.1 对话模型微调

*   **目标：** 使预训练模型具备良好的对话能力。
*   **数据：** 对话历史-回复对。
*   **挑战：** 保持对话的连贯性、信息性、安全性和多样性。

### 6.2 任务特定模型微调

*   **目标：** 使模型在特定NLP任务（如文本摘要、机器翻译、问答）上达到最优性能。
*   **数据：** 任务相关的输入-输出对（如文章-摘要、问题-答案）。

### 6.3 领域适应

*   **目标：** 使通用模型适应特定领域（如医疗、法律、金融）。
*   **数据：** 领域相关的文本和标注数据。

## 7. 总结

*   **监督微调**（SFT）是在预训练模型基础上，利用带标签数据进行进一步训练的过程，目的是使模型适应特定任务或指令。
*   SFT是连接通用预训练模型和特定应用场景的桥梁，对于提升模型的实用性和性能至关重要。
*   SFT的主要挑战包括**数据质量与数量**、**过拟合**和**灾难性遗忘**。
*   **指令微调**是SFT的重要应用，旨在提升模型的通用指令遵循能力。
*   SFT通常是**强化学习**（如RLHF）等更复杂训练流程的前置步骤。
*   理解SFT的概念、过程和挑战，是掌握大模型完整训练和应用流程的关键。
