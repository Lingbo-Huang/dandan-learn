# 参数高效微调

## 1. 引言

随着大语言模型（LLM）的规模呈指数级增长（从数十亿到数千亿参数），**全量参数微调**（Full Fine-Tuning）在计算资源、存储成本和部署复杂性方面面临严峻挑战。为了解决这些问题，**参数高效微调**（Parameter-Efficient Fine-Tuning, PEFT）技术应运而生，并迅速成为大模型适应下游任务的主流范式。

**参数高效微调**的核心思想是：在微调过程中，**冻结**（Freeze）预训练模型的绝大部分参数，只更新或引入**少量**的、**额外的**参数。这些少量参数能够“引导”或“调整”整个大模型的行为，使其适配特定的下游任务。

对于一个从传统软件开发转型的工程师来说，可以将PEFT类比为对一个庞大的软件库进行“插件式”定制，而不是修改其全部源代码。你开发一个轻量级的“插件”或“补丁”，这个插件通过钩子（Hooks）或接口与主程序交互，从而改变程序的行为，而主程序的大部分代码保持不变。

## 2. 参数高效微调的动机与优势

### 2.1 动机

*   **资源限制：** 全量微调大模型需要巨大的GPU内存和存储空间，对于大多数个人研究者和中小企业来说是不可行的。
*   **部署效率：** 为多个任务维护多个全量微调的模型副本成本高昂且管理复杂。
*   **知识保留：** 全量微调存在灾难性遗忘的风险，PEFT通过冻结大部分参数，更好地保留了预训练知识。

### 2.2 优势

1.  **资源效率高：**
    *   **内存占用低：** 只需计算和存储少量新增参数的梯度。
    *   **训练速度快：** 需要更新的参数少，前向/反向传播计算量小。
2.  **存储成本低：**
    *   **模型共享：** 预训练的主干模型可以被多个不同的PEFT模块共享。
    *   **增量存储：** 只需存储少量的增量参数，而不是完整的模型副本。
3.  **部署灵活：**
    *   **动态适配：** 可以在推理时动态加载不同的PEFT模块（如LoRA Adapter），实现一个主模型服务多种任务。
    *   **快速切换：** 任务切换只需更换轻量级的适配器，无需重新加载整个大模型。
4.  **缓解遗忘：** 冻结大部分预训练参数，有效缓解了灾难性遗忘问题。

## 3. 主要的PEFT方法

PEFT方法种类繁多，但其核心思想都是冻结主干模型，通过不同的机制引入少量可训练参数。以下是几种最具代表性和影响力的PEFT方法：

### 3.1 LoRA (Low-Rank Adaptation)

**核心思想：**
将预训练模型中权重矩阵的更新（微调过程中发生的改变）约束在一个**低秩**（Low-Rank）子空间内。具体来说，对于一个预训练权重矩阵 `W₀`，微调后的权重 `W` 被参数化为：
```
W = W₀ + ΔW
```
其中 `ΔW` 是更新量。LoRA假设 `ΔW` 可以被分解为两个更小矩阵的乘积：
```
ΔW = A * B
```
其中 `A` 的形状是 `(d, r)`，`B` 的形状是 `(r, k)`。`r`（秩）远小于 `d` 和 `k`。

**实现：**
*   在模型的某些层（通常是注意力权重和FFN权重）旁，**冻结**原始权重 `W₀`。
*   **添加**两个可训练的小矩阵 `A` 和 `B`。
*   前向传播时，实际计算的是 `W₀ * x + B * A * x`，其中 `B * A * x` 即为LoRA的调整项。
*   反向传播时，只计算 `A` 和 `B` 的梯度并更新它们。

**优点：**
*   **高效：** 极大地减少了可训练参数量（通常减少99%以上）。
*   **简单：** 实现相对简单，易于集成到现有模型中。
*   **有效：** 在许多任务上性能接近全量微调。

**缺点：**
*   推理时需要进行额外的矩阵乘法，略微增加延迟。

### 3.2 Adapter

**核心思想：**
在预训练模型的每一层（或某些层）中，**插入**一个小型的神经网络模块（Adapter Block），只训练这些Adapter模块的参数，而冻结原始模型的参数。

**结构：**
一个典型的Adapter模块包含以下组件：
1.  **下投影（Down-projection）：** 一个线性层，将输入维度从 `d` 降到一个较小的中间维度 `r` (e.g., `r = 64`)。
2.  **非线性激活函数：** 如ReLU或Swish。
3.  **上投影（Up-projection）：** 一个线性层，将维度从 `r` 恢复到 `d`。
4.  **残差连接（Residual Connection）：** 将Adapter的输出与原始输入相加。

**实现：**
*   在Transformer层的结构中（如在自注意力和FFN之后）插入Adapter模块。
*   训练时，冻结原始Transformer层的参数，只训练Adapter模块的参数。
*   推理时，数据流经原始层后，再流经Adapter模块，最后通过残差连接合并。

**优点：**
*   **模块化：** Adapter模块是独立的，易于添加或移除。
*   **任务特定：** 不同任务可以训练不同的Adapter模块。

**缺点：**
*   增加了模型的深度，可能略微影响推理速度。
*   性能可能略低于LoRA和全量微调。

### 3.3 Prefix-Tuning / Prompt Tuning

**核心思想：**
不直接修改模型的内部参数，而是优化输入到模型的**前缀**（Prefix）。这个前缀是一组可学习的向量，被添加到输入序列的开头或每个注意力层的Key和Value序列的开头。

**Prefix-Tuning：**
*   优化连续的、与具体词无关的向量前缀。
*   这些向量被拼接到每一层注意力机制的Key和Value序列的前面。

**Prompt Tuning：**
*   优化的是实际的、可读的词（或Token）作为前缀。
*   通常优化“软提示”（Soft Prompt），即词嵌入向量本身，而不是离散的词。

**优点：**
*   **非侵入性：** 不修改模型结构，只优化输入。
*   **高效：** 优化的参数量少。

**缺点：**
*   通常需要较长的前缀才能达到较好效果。
*   可解释性相对较差。

### 3.4 IA³ (Injective Adapter with Adaptive Activation)

**核心思想：**
一种更轻量级的Adapter变体。它不使用下投影-上投影的结构，而是为输入的每个维度学习一个**可学习的缩放因子**（Scaling Factor）。

**实现：**
*   对于输入 `h`，输出为 `h * s`，其中 `s` 是一个与 `h` 维度相同的可学习向量。
*   通常只在注意力的输出（After Attention）和FFN的输出（After FFN）处应用。
*   可以与LoRA等其他方法结合使用。

**优点：**
*   **参数极少：** 只需要学习与模型隐藏层维度相同的向量。
*   **计算简单：** 只是逐元素相乘。

## 4. PEFT方法的比较与选择

| 方法         | 可训练参数量 | 推理速度 | 实现复杂度 | 性能     | 适用场景                     |
| :----------- | :----------- | :------- | :--------- | :------- | :--------------------------- |
| **LoRA**     | 极少         | 略慢     | 中等       | 高       | 广泛适用，最流行             |
| **Adapter**  | 少量         | 略慢     | 中等       | 中等     | 需要模块化、多任务场景       |
| **Prefix**   | 少量         | 快       | 中等       | 中等     | 不想修改模型结构的场景       |
| **IA³**      | 极少         | 快       | 简单       | 中等     | 资源极度受限的场景           |

**选择建议：**
*   **LoRA** 是目前最推荐的默认选择，因为它在参数效率、实现复杂度和性能之间取得了很好的平衡。
*   如果需要严格的模块化和任务隔离，可以考虑 **Adapter**。
*   如果希望完全不修改模型内部结构，可以考虑 **Prefix-Tuning**。
*   如果资源极度受限，可以考虑 **IA³**。

## 5. PEFT在大模型中的应用

PEFT技术在现代大模型的训练和应用中扮演着至关重要的角色：

*   **模型服务：** 如Hugging Face的PEFT库，允许用户为一个大模型（如LLaMA2）训练多个LoRA Adapter，用于不同的应用（对话、翻译、摘要），并在服务时动态加载。
*   **研究与开发：** 使得研究人员能够在消费级GPU上对大模型进行微调实验。
*   **个性化：** 为不同用户或客户定制模型，只需训练和存储轻量级的Adapter，而无需为每个人维护一个完整模型。
*   **持续学习：** 通过添加新的Adapter模块来适应新任务，而不会干扰旧任务的性能。

## 6. 总结

*   **参数高效微调**（PEFT）是应对大模型全量微调资源瓶颈的关键技术。
*   其核心思想是冻结预训练模型的大部分参数，只训练或引入少量额外参数来适配下游任务。
*   **LoRA**、**Adapter**、**Prefix-Tuning** 和 **IA³** 是四种主流的PEFT方法，各有优缺点。
*   PEFT极大地降低了微调大模型的门槛，提高了部署效率和灵活性，是当前大模型应用的基石技术之一。
*   理解PEFT对于在资源受限环境下有效利用大模型至关重要。
