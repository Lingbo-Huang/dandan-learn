# 预训练任务

## 1. 引言

大语言模型（LLM）之所以能够展现出强大的语言理解和生成能力，其关键在于**预训练阶段**。在这一阶段，模型通过在大规模无标签文本数据上学习执行各种**预训练任务**（Pretraining Tasks），从而获得丰富的语言知识和通用的表示能力。

**预训练任务**是模型学习的“课程”或“练习题”。它们的设计至关重要，因为它们决定了模型将学习到什么样的知识和技能。好的预训练任务能够让模型：
*   学习到语言的**语法**、**语义**和**上下文依赖**关系。
*   捕捉到世界的**常识性知识**和**事实性知识**。
*   发展出**推理**、**归纳**和**泛化**的能力。

与传统的监督学习不同，预训练任务通常是**自监督**（Self-supervised）的，这意味着它们可以从原始数据本身构造出监督信号，而无需人工标注的标签。这使得利用互联网上海量的文本数据进行训练成为可能。

## 2. 自监督学习与预训练任务

### 2.1 什么是自监督学习？

**自监督学习**是一种机器学习范式，它从数据本身生成监督信号来学习表示。它不需要人工标注的标签，而是通过设计**辅助任务**（Pretext Tasks）来创造学习目标。

**核心思想：**
*   **破坏数据：** 以某种方式修改或破坏原始数据（如遮蔽词、打乱顺序）。
*   **重建数据：** 训练模型从被破坏的数据中恢复原始信息。
*   **学习表示：** 通过解决重建任务，模型被迫学习数据的内在结构和表示。

### 2.2 预训练任务与自监督学习的关系

在大语言模型的背景下，**预训练任务**就是**自监督学习**的具体实现。模型通过执行这些任务，从无标签文本中学习。

*   **输入：** 原始文本。
*   **构造的标签：** 通过特定规则从输入文本中自动生成（如被遮蔽的词、下一个词、句子关系等）。
*   **模型目标：** 预测这些自动生成的“标签”。

## 3. 经典的预训练任务

### 3.1 语言建模（Language Modeling）

这是最基础也是最重要的预训练任务，分为自回归语言建模和掩码语言建模。

#### 自回归语言建模（Autoregressive Language Modeling）

*   **任务描述：** 给定前面的词序列，预测下一个词。
*   **数学表达：** 学习 `P(w_t | w₁, ..., w_{t-1})`
*   **模型架构：** 通常使用**仅解码器**（Decoder-only）的Transformer架构。
*   **代表模型：** GPT系列、LLaMA系列。
*   **优势：** 天然适合文本生成任务，能学习到强大的序列生成能力。
*   **劣势：** 无法直接利用未来上下文信息，在某些理解任务上可能不如双向模型。

#### 掩码语言建模（Masked Language Modeling, MLM）

*   **任务描述：** 随机遮蔽输入文本中的一些词，训练模型预测这些被遮蔽的词。
*   **数学表达：** 学习 `P(w_t | w₁, ..., w_{t-1}, w_{t+1}, ..., w_T)` （对于被遮蔽的 `w_t`）
*   **模型架构：** 通常使用**仅编码器**（Encoder-only）的Transformer架构。
*   **实现细节（以BERT为例）：**
    *   随机选择15%的Token进行遮蔽。
    *   对于这15%的Token：
        *   80%替换为`[MASK]`。
        *   10%替换为随机Token。
        *   10%保持不变。
*   **代表模型：** BERT及其变体。
*   **优势：** 能利用双向上下文信息，对语言理解任务效果好。
*   **劣势：** 预训练和微调阶段存在`[MASK]`标记不匹配的问题；不适合直接生成文本。

### 3.2 下一句预测（Next Sentence Prediction, NSP）

*   **任务描述：** 给定两个句子A和B，判断B是否是A在原文中的下一个句子。
*   **标签：** 二分类标签（是/否）。
*   **目的：** 增强模型对句子间关系的理解能力。
*   **实现：**
    *   50%的时间，B是A的下一句（正例）。
    *   50%的时间，B是语料库中随机抽取的句子（负例）。
*   **模型输入：** `[CLS]` A `[SEP]` B `[SEP]`
*   **模型输出：** 基于`[CLS]`标记的表示进行分类。
*   **代表模型：** BERT（但后来发现其效果有限，RoBERTa等模型已弃用）。

### 3.3 句子排序（Sentence Ordering）

*   **任务描述：** 给定一个被打乱顺序的句子列表，恢复其正确的顺序。
*   **目的：** 训练模型理解长文本的连贯性和逻辑结构。
*   **优势：** 比NSP更能捕捉长距离的句子间依赖关系。

## 4. 高级预训练任务与目标

随着研究的深入，研究者们设计了更多样化、更复杂的预训练任务，以进一步提升模型能力。

### 4.1 对比学习（Contrastive Learning）

*   **核心思想：** 将语义相似的样本（正例）拉近，将语义不同的样本（负例）推远。
*   **应用：**
    *   **SimCSE：** 通过对同一个句子进行两次不同的Dropout，生成两个嵌入向量，将它们作为正例，其他句子的嵌入作为负例，进行对比学习。
    *   **Sentence-BERT：** 通过对三元组（Anchor, 正例, 负例）进行训练，学习句子嵌入。

### 4.2 去噪自编码（Denoising Autoencoding）

*   **任务描述：** 对输入文本施加更复杂的噪声（如删除、替换、打乱），然后训练模型恢复原始文本。
*   **代表模型：**
    *   **BART：** 使用任意噪声混合破坏文本，然后训练一个标准的Transformer编码器-解码器模型来重建原始文本。
    *   **T5：** 将各种NLP任务都统一为“文本到文本”的转换任务，包括去噪预训练。

### 4.3 前缀语言建模（Prefix Language Modeling）

*   **任务描述：** 结合了自回归和自编码的思想。模型可以看到部分未来的Token，但不是全部。
*   **实现：** 在训练时，使用一种特殊的注意力掩码，允许每个位置关注其左侧的一个区间和右侧的一个子区间。
*   **代表模型：** ELMo, XLNet。
*   **优势：** 试图结合双向上下文的好处和自回归生成的能力。

### 4.4 多任务预训练（Multi-Task Pretraining）

*   **任务描述：** 同时或交替地训练模型执行多种不同的预训练任务。
*   **目的：** 让模型学习到更全面、更鲁棒的语言表示。
*   **代表模型：**
    *   **T5：** 将所有任务（包括预训练和下游任务）都格式化为“文本到文本”的转换。
    *   **GLM：** 结合了自回归填充（Autoregressive Blank Infilling）和长文本建模。

### 4.5 替换Token检测（Replaced Token Detection, RTD）

*   **任务描述：** 生成器（Generator）模型首先预测被`[MASK]`替换的Token，然后判别器（Discriminator）模型判断输入Token是原始Token还是生成器生成的Token。
*   **目的：** 提高预训练效率，减少计算开销。
*   **代表模型：** ELECTRA。
*   **优势：** 比MLM更高效，因为每个位置都需要预测（而不是只预测被遮蔽的位置）。

## 5. 预训练任务设计的原则与考量

### 5.1 任务复杂度

*   任务既不能太简单（模型学不到有用信息），也不能太难（模型无法学会）。
*   任务应该能够促使模型学习到对下游任务有用的表示。

### 5.2 计算效率

*   预训练通常需要巨大的计算资源。设计高效的预训练任务可以显著降低训练成本。

### 5.3 与下游任务的相关性

*   理想情况下，预训练任务应与模型预期应用的下游任务相关，以便更好地进行知识迁移。

### 5.4 可扩展性

*   任务应能随着数据量和模型规模的增加而持续提升性能。

## 6. 预训练任务的演变趋势

1.  **从单一到多样：** 从最初的单一MLM或LM任务，发展到结合多种任务的多任务学习。
2.  **从简单到复杂：** 从简单的词预测，到句子关系判断，再到复杂的文本重建。
3.  **从固定到自适应：** 从固定的任务设计，到根据模型状态或数据动态调整任务。
4.  **从模型内到模型间：** 从单个模型执行预训练任务，到多个模型（如生成器和判别器）协同进行预训练（如ELECTRA）。

## 7. 总结

*   **预训练任务**是大语言模型获得强大能力的核心驱动力，它们是自监督学习的具体体现。
*   **经典任务**包括自回归语言建模（GPT类）和掩码语言建模（BERT类），分别侧重于生成和理解。
*   **高级任务**如对比学习、去噪自编码、多任务学习等，进一步提升了模型的性能和效率。
*   **任务设计**需要考虑复杂度、效率、相关性和可扩展性。
*   理解不同的预训练任务有助于选择合适的模型架构和训练策略，对于应用大模型解决实际问题至关重要。
