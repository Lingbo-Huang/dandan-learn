# Transformer架构

## 1. 引言

在深度学习领域，尤其是自然语言处理（NLP）和大语言模型（LLM）的发展中，**Transformer架构**无疑是一座里程碑式的丰碑。它由Vaswani等人在2017年的论文《Attention is All You Need》中首次提出，彻底改变了序列建模的方式。

在Transformer出现之前，循环神经网络（RNN）及其变体（如LSTM、GRU）是处理序列数据的主流方法。然而，RNN存在固有的缺陷：**序列依赖性**（无法并行化）和**长距离依赖建模困难**（梯度消失/爆炸）。Transformer通过引入**自注意力机制**（Self-Attention），一举解决了这些问题，为训练更大、更强的模型铺平了道路。

对于大语言模型而言，无论是GPT系列的**自回归**（Decoder-only）架构，还是BERT系列的**自编码**（Encoder-only）架构，其核心计算单元都源于或基于Transformer。因此，深入理解Transformer架构是掌握大模型核心技术的关键。

## 2. Transformer的整体结构

标准的Transformer模型由**编码器-解码器**（Encoder-Decoder）结构组成，专门用于解决序列到序列（Seq2Seq）任务，如机器翻译。

### 2.1 编码器（Encoder）

编码器由 `N` 个相同的层（Layer）堆叠而成（原论文中 `N=6`）。每一层包含两个主要子层：

1.  **多头自注意力子层（Multi-Head Self-Attention Sublayer）：**
    *   接收来自前一层的表示（第一层接收词嵌入）。
    *   通过自注意力机制，使序列中的每个位置都能关注到序列的其他所有位置，捕获全局依赖关系。
    *   通常后接**残差连接**（Residual Connection）和**层归一化**（Layer Normalization）。

2.  **位置前馈网络子层（Position-wise Feed-Forward Networks Sublayer）：**
    *   这是一个两层的全连接前馈网络，独立地应用于每个位置。
    *   `FFN(x) = max(0, xW₁ + b₁)W₂ + b₂` （通常第一层维度较大，如2048或4096）。
    *   同样后接残差连接和层归一化。

### 2.2 解码器（Decoder）

解码器也由 `N` 个相同的层堆叠而成。每一层包含三个主要子层：

1.  **掩码多头自注意力子层（Masked Multi-Head Self-Attention Sublayer）：**
    *   与编码器的自注意力类似，但为了防止在训练时位置 `i` 关注到后续位置（`i+1, i+2, ...`）的信息（这在生成时是不可见的），需要对注意力分数矩阵进行**掩码**（Masking）。

2.  **多头编码器-解码器注意力子层（Multi-Head Encoder-Decoder Attention Sublayer）：**
    *   这是连接编码器和解码器的关键。`Q` 来自解码器的前一层输出，`K` 和 `V` 来自**整个**编码器的最终输出。
    *   使得解码器的每个位置都能关注到输入序列的所有位置。

3.  **位置前馈网络子层（Position-wise Feed-Forward Networks Sublayer）：**
    *   与编码器中的FFN相同。

所有子层的输出都经过残差连接和层归一化。

### 2.3 输入与输出

*   **输入：** 源序列（如英文句子）的词嵌入（Token Embeddings）序列。为了注入序列顺序信息，会将**位置编码**（Positional Encoding）添加到词嵌入中。
*   **输出：** 目标序列（如法文句子）的词嵌入序列。通过一个线性层和Softmax层，将解码器的输出转换为词汇表上的概率分布，用于预测下一个词。

## 3. 核心组件：注意力机制（Attention Mechanism）

### 3.1 从RNN到Attention

在传统的Seq2Seq模型（如RNN Encoder-Decoder）中，编码器将整个输入序列压缩成一个**固定长度的上下文向量**（Context Vector），解码器再基于这个向量生成输出序列。当输入序列很长时，这个固定向量难以有效承载所有信息，导致性能瓶颈。

**注意力机制**的核心思想是：解码器在生成每个输出词时，不应该只关注一个固定的上下文向量，而应该根据当前已生成的部分和即将生成的目标，**有选择地关注**输入序列的不同部分。

### 3.2 缩放点积注意力（Scaled Dot-Product Attention）

这是Transformer中使用的注意力计算方式。

**输入：**
*   **查询**（Query, Q）：形状 `(L_q, d_k)`
*   **键**（Key, K）：形状 `(L_k, d_k)`
*   **值**（Value, V）：形状 `(L_k, d_v)`
*   `L_q` 是查询序列的长度，`L_k` 是键/值序列的长度，`d_k` 和 `d_v` 是特征维度。

**计算步骤：**
1.  **计算注意力分数（Compatibility）：** 计算Query和Key之间的相似度。Transformer使用**点积**（Dot Product）。
    ```
    Scores = Q * K^T  // 形状 (L_q, L_k)
    ```
2.  **缩放（Scaling）：** 为了稳定梯度，将点积结果除以 `sqrt(d_k)`。
    ```
    Scaled_Scores = Scores / sqrt(d_k)
    ```
3.  **Softmax归一化：** 对每一行（对应一个查询）进行Softmax，得到注意力权重（Attention Weights），表示对每个键值对的关注程度。
    ```
    Attention_Weights = Softmax(Scaled_Scores) // 形状 (L_q, L_k)
    ```
4.  **加权求和（Output）：** 使用注意力权重对Value进行加权求和，得到最终的注意力输出。
    ```
    Output = Attention_Weights * V // 形状 (L_q, d_v)
    ```

**数学公式：**
```
Attention(Q, K, V) = Softmax(Q * K^T / sqrt(d_k)) * V
```

**为什么需要缩放？**
当 `d_k` 较大时，`Q*K^T` 的点积值也会变得很大，这会导致Softmax函数的输入进入饱和区（梯度极小），使得模型训练困难。除以 `sqrt(d_k)` 可以将方差稳定在1左右，缓解这个问题。

### 3.3 多头注意力（Multi-Head Attention）

为了允许模型关注不同位置的不同表示子空间的信息，Transformer使用了**多头注意力**。

**过程：**
1.  将输入的 `Q`, `K`, `V` 通过不同的、可学习的线性变换（权重矩阵 `W_i^Q`, `W_i^K`, `W_i^V`）映射到 `h` 个不同的低维子空间（`h` 是头数）。
2.  在每个子空间 `i` 上，并行地执行一次缩放点积注意力，得到输出 `head_i`。
3.  将 `h` 个 `head_i` 拼接（Concatenate）起来。
4.  再通过一个线性变换（权重矩阵 `W^O`）得到最终的多头注意力输出。

**数学公式：**
```
MultiHead(Q, K, V) = Concat(head_1, head_2, ..., head_h) * W^O
其中 head_i = Attention(Q * W_i^Q, K * W_i^K, V * W_i^V)
```

**优势：**
*   允许模型在不同的表示子空间中并行地关注信息。
*   提供了多个“表示子空间”（representation subspaces）的视角，增强了模型的表示能力。

## 4. 位置编码（Positional Encoding）

Transformer的注意力机制本身是**排列不变**（Permutation-Invariant）的，即输入序列中词的顺序被打乱，注意力的计算结果不会改变。然而，对于自然语言这样的序列数据，**顺序信息至关重要**。

因此，必须向模型提供序列中词的**位置信息**。Transformer通过**位置编码**来实现这一点。

### 4.1 为什么需要位置编码？

*   RNN天然具有顺序性，当前时刻的隐藏状态是前一时刻状态的函数。
*   CNN通过滑动窗口捕获局部顺序。
*   纯注意力机制没有内在的顺序感。

### 4.2 Transformer中的位置编码

原论文提出了两种位置编码方法：

1.  **可学习的位置编码（Learned Positional Encoding）：**
    *   为每个位置 `pos` (0到最大序列长度) 和每个维度 `i` (0到 `d_model`) 学习一个编码向量。
    *   这些编码向量和词嵌入一样，通过训练来优化。

2.  **固定的正弦位置编码（Sinusoidal Positional Encoding）：**
    *   使用不同频率的正弦和余弦函数来生成位置编码。
    *   对于位置 `pos` 和维度 `i`：
        ```
        PE_(pos, 2i)     = sin(pos / 10000^(2i/d_model))
        PE_(pos, 2i+1)   = cos(pos / 10000^(2i/d_model))
        ```
    *   **优点：**
        *   能够扩展到比训练时更长的序列长度。
        *   为模型提供了关于相对位置和绝对位置的信息。对于任何固定的偏移量 `k`，`PE_{pos+k}` 可以表示为 `PE_{pos}` 的线性函数。

在实际应用中，这两种方法的性能差别不大。现代实现中，可学习的位置编码更为常见。

## 5. Transformer的变体与在大模型中的应用

标准的Encoder-Decoder Transformer主要用于机器翻译。在大语言模型中，根据任务需求，通常会使用其变体：

### 5.1 仅编码器（Encoder-Only）架构 - 自编码模型

*   **代表：** BERT及其衍生模型。
*   **结构：** 堆叠的Transformer编码器层。
*   **注意力：** 双向自注意力（无遮蔽）。
*   **应用：** 文本理解任务，如分类、问答、命名实体识别等。

### 5.2 仅解码器（Decoder-Only）架构 - 自回归模型

*   **代表：** GPT系列、LLaMA系列。
*   **结构：** 堆叠的Transformer解码器层，但去掉了编码器-解码器注意力子层。
*   **注意力：** 因果（单向）自注意力（带掩码）。
*   **应用：** 文本生成任务。

### 5.3 编码器-解码器（Encoder-Decoder）架构

*   **代表：** T5, BART。
*   **结构：** 完整的Transformer Encoder-Decoder结构。
*   **应用：** 需要将一种形式的文本转换为另一种形式的任务，如翻译、摘要、问答等。

## 6. Transformer的优势与局限性

### 6.1 优势

1.  **并行化：** 与RNN不同，Transformer不需要按顺序处理序列元素，可以高度并行化，训练速度大大加快。
2.  **长距离依赖：** 注意力机制允许模型直接关注序列中任意两个位置的关系，有效捕捉长距离依赖，避免了RNN的梯度消失问题。
3.  **强大的建模能力：** 在机器翻译、文本生成等任务上取得了SOTA（State-of-the-Art）性能。
4.  **可解释性：** 注意力权重可以提供一定程度的模型决策解释。

### 6.2 局限性

1.  **计算与内存开销：** 对于长度为 `n` 的序列，自注意力的计算复杂度和内存需求都是 `O(n²)`。对于很长的序列，这会成为瓶颈。
2.  **缺乏归纳偏置：** 相比CNN（平移不变性）和RNN（时间不变性），Transformer的归纳偏置较少，可能需要更多的数据来学习。
3.  **位置编码的局限：** 固定的位置编码可能无法很好地泛化到训练时未见的长序列。

## 7. 总结

*   **Transformer** 是一种基于注意力机制的深度学习架构，摒弃了RNN和CNN，实现了高度并行化。
*   **缩放点积注意力** 是其核心计算单元，通过 `Softmax(Q*K^T/sqrt(d_k))*V` 计算。
*   **多头注意力** 允许模型在不同子空间并行关注信息，增强了表示能力。
*   **位置编码** 被添加到词嵌入中，以向模型提供序列顺序信息。
*   **残差连接** 和 **层归一化** 被应用于每个子层，有助于训练深层网络。
*   Transformer有三种主要变体：**仅编码器**（BERT类）、**仅解码器**（GPT类）、**编码器-解码器**（T5类），分别适用于不同的任务。
*   Transformer是现代大语言模型（如BERT、GPT、LLaMA）的基石，对深度学习领域产生了深远影响。
