# 自回归模型

## 1. 引言

在大语言模型（LLM）的预训练阶段，**自回归模型**（Autoregressive Model）是构建生成式语言模型的主流范式。理解自回归模型对于掌握大模型的核心工作机制至关重要。

**核心思想：**
自回归模型通过学习**预测序列中的下一个元素**来训练。在自然语言处理中，这意味着模型学习根据前面的词来预测下一个词。这种训练方式使模型能够捕捉语言的复杂结构、语法和语义，并最终具备生成连贯、流畅文本的能力。

**类比：**
可以把自回归模型想象成一个逐字填空的游戏高手。给定一个不完整的句子，它能根据已有的上下文，预测出下一个最可能出现的字或词。通过不断地进行这样的预测和学习，它逐渐掌握了语言的规律。

## 2. 自回归模型的定义与数学表达

### 2.1 基本定义

自回归模型是一种**生成模型**，它将联合概率分布 `P(x₁, x₂, ..., x_T)` 分解为一系列**条件概率**的乘积。对于一个包含 `T` 个元素的序列 `x = (x₁, x₂, ..., x_T)`，自回归模型假设：

```
P(x₁, x₂, ..., x_T) = Π (t=1 to T) P(x_t | x₁, x₂, ..., x_{t-1})
```

其中 `P(x_t | x₁, x₂, ..., x_{t-1})` 是在给定序列前 `t-1` 个元素的条件下，预测第 `t` 个元素的概率分布。

### 2.2 在自然语言中的应用

在自然语言处理中，序列 `x` 通常是**词序列**（Token Sequence） `w = (w₁, w₂, ..., w_T)`。因此，语言模型的目标是学习：

```
P(w₁, w₂, ..., w_T) = Π (t=1 to T) P(w_t | w₁, w₂, ..., w_{t-1})
```

**训练目标：**
最大化训练语料库中所有句子（或文本片段）的对数似然（Log-Likelihood）：

```
L = Σ log P(w_t | w₁, ..., w_{t-1})
```

这等价于最小化**负对数似然**（Negative Log-Likelihood），也就是**交叉熵损失**。

## 3. 自回归模型的工作机制

### 3.1 训练阶段（Pre-training）

1.  **数据准备：** 收集大规模文本数据集（如网页、书籍、维基百科等）。
2.  **Tokenization：** 将文本分割成模型能理解的单位（Token），如子词（Subword）、字（Character）或词（Word）。
3.  **任务设定：** 模型接收一段文本序列作为输入 `x = (x₁, x₂, ..., x_{t-1})`，目标是预测下一个Token `x_t`。
4.  **模型结构：** 通常使用**仅解码器**（Decoder-only）的Transformer架构。这种结构中的注意力机制是**因果性**（Causal）的，即每个位置只能关注到它之前的Token，防止信息泄露。
5.  **参数学习：** 通过反向传播和梯度下降算法，调整模型参数以最小化预测误差（交叉熵损失）。

**例子：**
输入文本："The cat sat on the"
模型的目标是最大化 `P("mat" | "The cat sat on the")` 的概率。

### 3.2 推理/生成阶段（Inference/Generation）

训练完成后，模型可以用于文本生成：

1.  **给定前缀：** 提供一个初始的文本前缀（Prompt）。
2.  **迭代生成：**
    *   模型根据前缀预测下一个Token的概率分布。
    *   根据这个分布采样或选择一个Token作为输出。
    *   将新生成的Token添加到输入序列中。
    *   重复此过程，直到生成结束标记（如 `<EOS>`）或达到最大长度。

**解码策略：**
*   **贪婪搜索（Greedy Search）：** 每次都选择概率最高的Token。
*   **束搜索（Beam Search）：** 维护多个候选序列，选择整体概率最高的序列。
*   **采样（Sampling）：** 根据概率分布随机采样，可以引入温度参数（Temperature）来控制随机性。

## 4. 自回归模型与自编码模型的对比

| 特性         | 自回归模型 (Autoregressive)             | 自编码模型 (Autoencoder)                     |
| :----------- | :-------------------------------------- | :------------------------------------------- |
| **方向性**   | 单向（通常从左到右）                    | 双向                                         |
| **训练任务** | 预测下一个Token                         | 重建被掩码/损坏的Token                       |
| **代表模型** | GPT系列, LLaMA系列                      | BERT, RoBERTa                                |
| **主要应用** | 文本生成                                | 理解任务（分类、问答、NER等）                |
| **优势**     | 生成能力强，能很好地建模Token间的依赖关系 | 能利用双向上下文信息，对理解任务效果好       |
| **劣势**     | 无法利用未来信息，理解任务可能不如双向模型 | 不能直接用于生成任务（需要额外的解码器结构） |

## 5. 自回归模型的优缺点

### 5.1 优点

1.  **强大的生成能力：** 天然适合文本生成任务，能够产生连贯、流畅的文本。
2.  **明确的学习目标：** 预测下一个Token是一个清晰、具体的目标，便于模型学习。
3.  **能捕捉长距离依赖：** 基于Transformer的现代自回归模型（如GPT）能够有效捕捉长距离的依赖关系。
4.  **应用广泛：** 可用于文本补全、对话、故事生成、代码生成等多种任务。

### 5.2 缺点

1.  **训练效率：** 由于是串行预测，无法像掩码语言模型那样并行化训练所有Token，训练速度相对较慢。
2.  **暴露偏差（Exposure Bias）：** 在训练时，模型总是基于真实的前序Token进行预测；而在推理时，它基于自己生成的（可能不完美）Token进行预测。这种不匹配可能导致错误在生成过程中累积。
3.  **缺乏双向上下文：** 标准的从左到右自回归模型在预测某个Token时，无法直接看到它右侧的上下文信息，这在某些理解任务中可能是一个劣势。

## 6. 典型的自回归模型架构

### 6.1 GPT系列

*   **GPT (Generative Pre-trained Transformer)：** 由OpenAI提出，是最早成功应用Transformer架构进行自回归语言建模的模型之一。
*   **GPT-2, GPT-3, GPT-3.5, GPT-4：** 后续版本通过不断增加模型规模和数据量，在各种NLP任务和生成任务上取得了显著的性能提升。
*   **架构：** 纯解码器（Decoder-only）结构，使用因果注意力（Causal Attention Mask）。

### 6.2 LLaMA系列

*   **LLaMA (Large Language Model Meta AI)：** 由Meta开发，旨在提供高效、开源的大型语言模型。
*   **LLaMA 2, LLaMA 3：** 后续版本在性能和效率上进一步优化。
*   **架构：** 同样是解码器-only结构。

### 6.3 其他模型

*   **OPT (Open Pre-trained Transformer)：** Meta开源的类似GPT的模型。
*   **BLOOM：** BigScience社区开源的多语言大模型。
*   **Falcon：** TII开源的高性能大模型。

## 7. 总结

*   **自回归模型**通过学习预测序列中的下一个元素来进行训练，是大语言模型预训练的主流范式之一。
*   其数学基础是将序列的联合概率分解为一系列条件概率的乘积。
*   训练时通过最大化下一个Token的预测概率来学习参数，推理时通过迭代预测来生成文本。
*   相比自编码模型（如BERT），它更擅长生成任务，但在某些理解任务上可能存在劣势。
*   GPT系列和LLaMA系列是自回归模型的典型代表，它们都基于Transformer的解码器-only架构。
*   理解自回归模型是掌握大模型预训练、微调和应用的基础。
