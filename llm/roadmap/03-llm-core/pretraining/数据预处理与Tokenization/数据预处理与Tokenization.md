# 数据预处理与Tokenization

## 1. 引言

在大语言模型（LLM）的预训练过程中，**数据预处理**（Data Preprocessing）和**Tokenization**（分词/标记化）是至关重要但常常被低估的步骤。原始文本数据是无法直接输入到神经网络中的，必须经过一系列的处理和转换，才能变成模型可以理解和学习的数字向量。

**数据预处理**涵盖了从原始文本清洗到最终模型输入格式化的整个流程，而**Tokenization**则是这个流程中最核心的环节，它决定了模型如何“看待”和处理语言的基本单位。

可以将Tokenization想象成教一个外国小朋友学中文。你不能直接把一整篇中文文章塞给他，而是需要先把文章分解成他能理解的单位，比如字、词或短语（Token），然后为每个单位分配一个唯一的ID（就像他的词汇表），最后再把这些ID序列输入到他的大脑（模型）中进行学习。

## 2. 数据预处理流程

在进行Tokenization之前，通常需要对原始文本数据进行一系列的清洗和预处理。

### 2.1 文本清洗（Text Cleaning）

原始文本数据往往包含各种噪声和不规范的格式，需要进行清洗以提高数据质量。

*   **去除或标准化特殊字符：** 处理HTML标签、XML标记、URL、邮箱地址等。
*   **处理空白字符：** 统一不同类型的空格、换行符、制表符等。
*   **大小写处理：** 根据模型设计决定是否进行大小写归一化（如全部转为小写）。
*   **去除多余符号：** 删除或替换连续的标点符号、特殊符号等。
*   **语言识别与过滤：** 对于多语言语料库，可能需要识别并过滤掉非目标语言的文本。

### 2.2 文本规范化（Text Normalization）

将文本转换为一种更标准、更一致的形式。

*   **数字规范化：** 将数字转换为统一的表示形式（如全部转为阿拉伯数字）。
*   **日期和时间规范化：** 统一日期和时间的表示格式。
*   **缩写展开：** 将缩写词展开为完整形式（如 "don't" -> "do not"）。

### 2.3 数据去重与过滤（Deduplication and Filtering）

*   **文档级去重：** 去除完全相同的文档。
*   **行级去重：** 去除完全相同的行或句子。
*   **低质量内容过滤：** 过滤掉过短、过长、内容过于简单或包含过多错误的文档。
*   **隐私信息过滤：** 移除可能包含个人身份信息（PII）的内容。

## 3. Tokenization基础概念

### 3.1 什么是Token？

**Token**是模型处理文本时的基本单位。它可以是一个词、一个子词（Subword）、一个字符，甚至是一个标点符号。Token的选择直接影响模型的性能、效率和泛化能力。

### 3.2 什么是Tokenization？

**Tokenization**是将原始文本字符串分割成一个**Token序列**的过程。这个过程通常包括两个步骤：
1.  **分割（Splitting）：** 将文本切分为一个个独立的单元（Token）。
2.  **映射（Mapping）：** 将每个Token映射到一个唯一的数字ID（Token ID），这个ID是模型实际处理的对象。

### 3.3 词汇表（Vocabulary）

**词汇表**（Vocabulary）是模型能识别的所有Token的集合。每个Token在词汇表中都有一个唯一的索引（Index/ID）。词汇表的大小是一个重要的超参数。

*   **大词汇表：** 可以更精确地表示文本，但会增加模型参数量和计算开销。
*   **小词汇表：** 模型更紧凑高效，但可能导致更多Token被拆分为未知词（UNK）或子词。

## 4. 常见的Tokenization方法

### 4.1 字符级（Character-Level）Tokenization

*   **定义：** 将文本分割成单个字符。
*   **优点：**
    *   词汇表非常小（通常几十到几百个，取决于语言）。
    *   不会出现未知词（UNK）问题。
    *   能自然处理拼写错误和新词。
*   **缺点：**
    *   序列长度会变得非常长，增加了模型的计算负担。
    *   模型需要学习字符之间的组合规律才能理解词义，学习难度大。
*   **应用：** 早期的神经语言模型、拼写检查、某些特定任务。

### 4.2 词级（Word-Level）Tokenization

*   **定义：** 将文本分割成单词。
*   **优点：**
    *   直观，符合人类对语言的理解。
    *   序列长度相对较短。
*   **缺点：**
    *   词汇表可能非常大，尤其是在处理多种语言或包含大量专有名词的文本时。
    *   无法很好地处理未登录词（Out-Of-Vocabulary, OOV）问题。通常会用一个特殊的`<UNK>`标记来表示所有未见过的词。
    *   对形态丰富的语言（如德语、芬兰语）效果不佳，因为这些语言有大量词形变化。
*   **实现：**
    *   **空格分词：** 简单地按空格和标点分割（适用于英语等以空格分词的语言）。
    *   **使用NLP库：** 如NLTK、spaCy提供的分词器，可以处理更复杂的分词规则。

### 4.3 子词级（Subword-Level）Tokenization

为了解决词级分词的OOV问题和字符级分词的长序列问题，子词级分词应运而生。它将词拆分成更小的、常见的子单元。

#### Byte-Pair Encoding (BPE)

*   **原理：** 一种基于频率的无损数据压缩算法，被巧妙地应用于分词。
*   **训练过程：**
    1.  初始化：将每个词拆分成字符序列，并在末尾加上结束符`</w>`。
    2.  统计：计算所有相邻子词对的频率。
    3.  合并：将频率最高的子词对合并成一个新的子词。
    4.  重复：重复步骤2和3，直到达到预设的词汇表大小或迭代次数。
*   **分词过程：** 对于一个新词，贪心地应用训练好的合并规则，将词拆分成已知的子词单元。
*   **优点：**
    *   有效地平衡了词汇表大小和序列长度。
    *   能较好地处理OOV问题，因为罕见词会被拆分成常见的子词。
*   **代表模型：** GPT-2, RoBERTa, LLaMA。

#### WordPiece

*   **原理：** 与BPE类似，但优化目标不同。它每次选择能最大化训练数据似然的子词对进行合并。
*   **优化目标：**
    ```
    目标 = argmax log p(x) = argmax Σ log p(token_i | token_1, ..., token_{i-1})
    ```
    通过选择合并后能最大化这个目标的子词对。
*   **代表模型：** BERT, DistilBERT, Electra。

#### Unigram Language Model

*   **原理：** 假设每个词都由子词序列生成，通过优化子词序列的概率来学习词汇表。
*   **过程：**
    1.  从一个大的初始词汇表（如所有字符和常用子词）开始。
    2.  使用EM算法或类似方法，迭代地评估移除每个子词对模型性能的影响。
    3.  移除对性能影响最小的子词，直到达到目标词汇表大小。
*   **代表模型：** Albert, T5, XLNet。

### 4.4 SentencePiece

*   **定义：** 一种无监督的文本分词工具，不依赖于空格，可以应用于任何语言。
*   **特点：**
    *   可以直接在原始文本（UTF-8 bytes）上训练，无需预处理。
    *   支持BPE和Unigram两种算法。
    *   产生的模型是一个独立的文件，便于分发和使用。
*   **优势：** 对多语言支持好，处理无空格分隔的语言（如中文、日文）更有效。

## 5. 特殊Token（Special Tokens）

在Tokenization过程中，除了表示文本内容的Token外，还需要一些**特殊Token**来指导模型的行为和结构。

*   **`[PAD]` / `<pad>`：** 用于填充（Padding），使一个批次（Batch）中的所有序列长度一致。
*   **`[UNK]` / `<unk>`：** 用于表示词汇表中不存在的词（未知词）。
*   **`[CLS]`：** 在BERT等模型中，放在序列开头，其最终隐藏状态常用于分类任务。
*   **`[SEP]`：** 在BERT等模型中，用于分隔两个句子或标记序列的结尾。
*   **`[MASK]`：** 在BERT等模型中，用于替换被遮蔽的词。
*   **`<s>` / `</s>`：** 在某些模型（如GPT, LLaMA）中，用于标记序列的开始和结束。

## 6. Tokenization在大模型中的实践

### 6.1 GPT/LLaMA系列（基于BPE）

*   使用**BPE**算法进行子词分词。
*   词汇表通常包含10,000到100,000个Token。
*   通常区分大小写。
*   使用`<|endoftext|>`作为文档分隔符和结束符。

### 6.2 BERT系列（基于WordPiece）

*   使用**WordPiece**算法进行子词分词。
*   词汇表大小通常为30,000左右。
*   对输入进行小写化处理（对于`bert-base-uncased`）。
*   使用`[CLS]`、`[SEP]`、`[MASK]`等特殊Token。

### 6.3 T5系列（基于SentencePiece - Unigram）

*   使用**SentencePiece**的**Unigram**模式。
*   词汇表大小通常为32,000。
*   不区分大小写（通常转为小写）。
*   使用`</s>`作为序列结束符。

## 7. Tokenization的挑战与考量

### 7.1 语言特异性

*   不同语言的分词策略差异很大。例如，中文、日文没有天然的词边界，需要专门的分词工具；而英语等语言则可以基于空格分词。

### 7.2 词汇表大小与效率

*   词汇表越大，模型表达能力越强，但参数量和计算开销也越大。
*   需要在模型性能和计算效率之间找到平衡。

### 7.3 OOV问题

*   如何优雅地处理训练时未见过的词，是一个持续的挑战。

### 7.4 领域适应

*   通用的分词器在特定领域（如医学、法律）可能表现不佳，可能需要领域特定的分词器。

## 8. 总结

*   **数据预处理**和**Tokenization**是大语言模型预训练的基础，直接影响模型的输入质量和学习效果。
*   Tokenization是将文本转换为模型可处理的Token序列的过程，涉及分割和映射两个步骤。
*   常见的Tokenization方法包括**字符级**、**词级**和**子词级**（BPE, WordPiece, Unigram）。
*   **子词级分词**（尤其是BPE和WordPiece）因其在处理OOV问题和平衡词汇表大小方面的优势，成为现代大模型的主流选择。
*   **特殊Token**在模型结构和任务执行中扮演关键角色。
*   不同的模型架构和训练目标会选择不同的Tokenization方法和特殊Token集合。
*   理解Tokenization对于模型训练、微调和推理都至关重要。
