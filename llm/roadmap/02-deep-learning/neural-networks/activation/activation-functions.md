# 激活函数

## 1. 引言

在前两节中，我们学习了感知机、神经网络的基本结构以及前向传播过程。我们了解到，神经元接收来自前一层的输入，进行加权求和并加上偏置，得到一个线性组合 `z = Wx + b`。然而，如果神经网络仅仅执行线性变换，无论网络有多少层，其整体仍然是一个线性模型，只能学习线性关系，无法解决复杂的非线性问题（如XOR）。

**激活函数**（Activation Function）是神经网络中不可或缺的组成部分，它被应用于神经元的线性输出 `z` 上，引入**非线性**，使得神经网络能够学习和表示复杂的非线性关系。可以说，没有激活函数，深度学习就失去了其强大的表达能力。

从生物学角度看，激活函数模拟了生物神经元的“激活”或“ firing ”行为，即当输入信号达到一定阈值时，神经元才会产生输出脉冲。

## 2. 激活函数需要满足的性质

一个优秀的激活函数通常需要满足以下性质：

1.  **非线性（Non-linearity）：** 这是最重要的性质。只有非线性激活函数才能使多层网络具有强大的表达能力。
2.  **可微性（Differentiability）：** 大多数优化算法（如梯度下降）依赖于导数（梯度）来更新参数。因此，激活函数最好是处处可微的，或者至少在大部分区域可微。在不可微点（如ReLU在0处），通常约定其导数为0或左/右导数。
3.  **计算简单（Computational Efficiency）：** 函数本身及其导数的计算应尽可能简单快速，以提高训练和推理速度。
4.  **非饱和性（Non-saturating）：** 当神经元的输入值非常大或非常小时，如果激活函数的导数趋近于0，就会导致梯度消失，使得训练变得缓慢甚至停滞。好的激活函数应尽量避免在大部分定义域内饱和。
5.  **输出范围（Output Range）：** 输出范围会影响网络的训练稳定性。例如，有界输出（如Sigmoid的(0,1)）可以将激活值规范化，但也可能导致梯度消失；无界输出（如ReLU的[0,+∞)）计算简单，但可能需要其他机制（如归一化）来稳定训练。

## 3. 常见的激活函数

### 3.1 Sigmoid函数

**定义：**
```
σ(z) = 1 / (1 + e^(-z))
```

**性质：**
*   **输出范围：** (0, 1)。这使得它非常适合用于二分类问题的输出层，可以解释为概率。
*   **单调性：** 单调递增。
*   **可微性：** 处处可微。
*   **导数：** `σ'(z) = σ(z) * (1 - σ(z))`。这个形式非常简洁，在反向传播中计算高效。

**优点：**
*   输出有界，使得神经元的激活相对“平滑”。
*   导数计算简单。

**缺点：**
*   **梯度消失（Vanishing Gradient）：** 当 `z` 的绝对值很大时，`σ'(z)` 会趋近于0。在深层网络中，这会导致梯度在反向传播时逐层衰减，训练变得非常缓慢。
*   **输出非零中心化（Non-zero-centered）：** Sigmoid函数的输出总是正数，这会导致下一层的神经元输入的权重更新方向趋于一致（要么都增加，要么都减少），可能影响收敛速度。
*   **计算复杂：** 涉及指数运算，计算量相对较大。

### 3.2 Tanh函数（双曲正切函数）

**定义：**
```
tanh(z) = (e^z - e^(-z)) / (e^z + e^(-z))
```
或者可以表示为Sigmoid的变换：
```
tanh(z) = 2 * σ(2z) - 1
```

**性质：**
*   **输出范围：** (-1, 1)。这是零中心化的。
*   **单调性：** 单调递增。
*   **可微性：** 处处可微。
*   **导数：** `tanh'(z) = 1 - (tanh(z))²`。

**优点：**
*   相比Sigmoid，它是**零中心化**的，通常训练效果更好。
*   导数计算简单。

**缺点：**
*   **梯度消失：** 与Sigmoid类似，当 `|z|` 很大时，导数也会趋近于0，存在梯度消失问题。
*   **计算复杂：** 同样涉及指数运算。

### 3.3 ReLU函数（Rectified Linear Unit）

**定义：**
```
ReLU(z) = max(0, z)
```

**性质：**
*   **输出范围：** [0, +∞)。
*   **单调性：** 单调递增。
*   **可微性：** 在 `z ≠ 0` 时处处可微。在 `z = 0` 处不可微，但通常约定其导数为0或1。
    *   `ReLU'(z) = 1` if `z > 0`
    *   `ReLU'(z) = 0` if `z < 0`
    *   `ReLU'(z) = 0` or `1` (约定) if `z = 0`

**优点：**
*   **计算极其简单：** 只需要一个阈值操作，计算速度非常快。
*   **有效缓解梯度消失：** 在 `z > 0` 的区域，导数恒为1，不会像Sigmoid和Tanh那样衰减。
*   **稀疏激活：** 一半的神经元（`z < 0` 时）会被抑制（输出为0），这被认为可以提高网络的表示效率和泛化能力。

**缺点：**
*   **死亡ReLU问题（Dying ReLU Problem）：** 在训练过程中，如果某个神经元的权重更新使得其在所有训练样本上的加权输入 `z` 都小于0，那么这个神经元将永远输出0，其梯度也将永远为0，无法再被激活。这个神经元就“死亡”了。
*   **输出非零中心化：** 输出范围是 [0, +∞)，不是零中心化的。

### 3.4 Leaky ReLU函数

为了解决ReLU的“死亡”问题，提出了Leaky ReLU。

**定义：**
```
LeakyReLU(z) = {
    z,      if z > 0
    αz,     if z ≤ 0
}
```
其中 `α` 是一个很小的常数（如0.01）。

**性质：**
*   **输出范围：** (-∞, +∞)。
*   **导数：**
    *   `LeakyReLU'(z) = 1` if `z > 0`
    *   `LeakyReLU'(z) = α` if `z < 0`
    *   `LeakyReLU'(z) = α` or `1` (约定) if `z = 0`

**优点：**
*   解决了ReLU的死亡问题，因为即使 `z < 0`，也有一个小的梯度 `α`，神经元不会完全“死亡”。
*   计算简单。

**缺点：**
*   `α` 的值需要手动设定，且效果可能不如自适应方法。

### 3.5 Parametric ReLU (PReLU) 和 Exponential Linear Unit (ELU)

这些是ReLU的进一步改进：

*   **PReLU：** 将Leaky ReLU中的 `α` 作为一个可学习的参数，让网络自己去决定负轴的斜率。
*   **ELU：** 在负轴上使用指数函数，更加平滑，并且在负值区域有非零均值，有助于提高学习速度。
    ```
    ELU(z) = {
        z,              if z > 0
        α(e^z - 1),     if z ≤ 0
    }
    ```

### 3.6 Swish函数

这是一个由Google研究人员提出的新激活函数，具有自门控（self-gated）的特性。

**定义：**
```
swish(z) = z * σ(z)
```
其中 `σ(z)` 是Sigmoid函数。

**优点：**
*   在实验中表现优于ReLU。
*   平滑且非单调。

**缺点：**
*   计算比ReLU复杂。

## 4. 如何选择激活函数？

*   **隐藏层：** ReLU是目前最广泛使用的默认选择，因为它计算简单且有效缓解了梯度消失问题。如果担心死亡ReLU问题，可以考虑Leaky ReLU或其变种（PReLU, ELU）。
*   **输出层：**
    *   **二分类：** Sigmoid，输出可以解释为概率。
    *   **多分类（C类）：** Softmax。Softmax函数将一个C维的实数向量转换为一个C维的概率分布向量。
        ```
        softmax(z_i) = e^{z_i} / Σ_{j=1}^{C} e^{z_j}
        ```
    *   **回归：** 通常不使用激活函数（即使用恒等函数 `f(z) = z`），或者根据输出范围选择（如Tanh用于(-1,1)范围）。

## 5. 总结

*   **激活函数**是神经网络引入非线性的关键，使深度模型能够学习复杂模式。
*   **Sigmoid**和**Tanh**是早期常用的激活函数，但由于梯度消失问题，在深层网络中表现不佳。
*   **ReLU**及其变种（Leaky ReLU, PReLU, ELU）因其计算简单和缓解梯度消失的特性，成为现代神经网络隐藏层的主流选择。
*   选择激活函数需要考虑其性质（非线性、可微性、计算复杂度、梯度流）以及具体的应用场景（隐藏层/输出层）。
*   理解激活函数对于理解网络的训练动态（如梯度消失/爆炸）和选择合适的网络架构至关重要。
