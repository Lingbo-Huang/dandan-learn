# 参数初始化

## 1. 引言

在训练神经网络之前，我们需要为网络中的所有参数（主要是权重 `W` 和偏置 `b`）设置初始值。这个过程称为**参数初始化**（Parameter Initialization）。

虽然这看起来是一个简单的步骤，但参数初始化对神经网络的训练过程和最终性能有着深远的影响。**不恰当的初始化**可能导致训练缓慢、难以收敛，甚至完全失败。

想象一下，如果把一个深度网络的所有权重都初始化为0，会发生什么？由于对称性，所有神经元在前向传播和反向传播时的行为都完全相同，网络无法打破这种对称性来学习到不同的特征。这就是一个极端但重要的例子，说明了初始化的重要性。

## 2. 初始化不当导致的问题

### 2.1 零初始化（Zero Initialization）

将所有权重初始化为0。

**问题：**
*   **对称性问题：** 对于同一层内的所有神经元，如果它们的权重完全相同，那么在前向传播时，它们会计算出相同的加权和 `z`，经过相同的激活函数后，输出 `a` 也相同。在反向传播时，它们会得到相同的梯度。这意味着无论训练多久，这些神经元都会保持完全同步，无法学习到不同的特征。网络的表达能力大大降低，等同于该层只有一个神经元。
*   **偏置项：** 偏置 `b` 可以初始化为0，因为它不涉及对称性问题。

### 2.2 过大初始化（Large Initialization）

将权重初始化为很大的随机值（例如，从均值为0，标准差为10的高斯分布中采样）。

**问题：**
*   **激活值爆炸：** 很大的权重会导致神经元的加权输入 `z` 很大，特别是当使用Sigmoid或Tanh激活函数时，这会使神经元的输出饱和在非线性区域的边缘（如Sigmoid的0或1）。在这些区域，激活函数的梯度非常小，接近于0。
*   **梯度消失：** 根据链式法则，很小的梯度在反向传播过程中会逐层相乘，导致靠近输入层的梯度指数级衰减，训练变得极其缓慢。

### 2.3 过小初始化（Small Initialization）

将权重初始化为非常小的随机值（例如，从均值为0，标准差为0.001的高斯分布中采样）。

**问题：**
*   **激活值消失：** 很小的权重会导致神经元的加权输入 `z` 接近于0，激活值也集中在激活函数的线性区域附近。
*   **梯度消失：** 同样，激活函数在线性区域的梯度也很小，反向传播时梯度会逐层衰减。

## 3. 理想的初始化目标

一个好的初始化策略应该努力实现以下目标：

1.  **打破对称性：** 同一层内的不同神经元需要有不同的初始权重，以便它们可以学习不同的特征。
2.  **保持方差稳定：** 在前向传播过程中，每一层的激活值（`a` 或 `z`）的方差不应爆炸式增长或指数级衰减。在反向传播过程中，梯度的方差也应保持稳定。这有助于信息和梯度在网络中有效地流动。
3.  **避免饱和：** 初始化应使神经元的初始加权输入 `z` 和激活输出 `a` 尽可能避免落在激活函数的饱和区域。

## 4. 常见的初始化方法

### 4.1 随机初始化（Random Initialization）

这是打破对称性的基本要求。通常从某个概率分布（如高斯分布或均匀分布）中采样来初始化权重。

*   **权重 `W`：** 通常从均值为0的分布中随机采样。
*   **偏置 `b`：** 通常初始化为0向量。

### 4.2 Xavier初始化（Glorot初始化）

由Xavier Glorot和Yoshua Bengio在2010年提出，主要针对Sigmoid或Tanh激活函数。

**核心思想：**
设计初始化分布，使得前向传播时每一层的激活值方差保持不变，反向传播时每一层的梯度方差也保持不变。

**假设：**
*   激活函数是线性的，或者在0附近近似线性。
*   权重 `W` 的初始化相互独立且与输入 `x` 独立。

**推导结果：**
为了保持方差稳定，需要：
```
Var(W^{[l]}) = 1 / n^{[l-1]}
```
其中 `n^{[l-1]}` 是第 `l-1` 层的神经元数量（即当前层的输入维度）。

**实现方式：**
*   从均匀分布 `U(-limit, limit)` 中采样，其中 `limit = √(6 / (n^{[l-1]} + n^{[l]}))`。
*   或从高斯分布 `N(0, σ²)` 中采样，其中 `σ² = 2 / (n^{[l-1]} + n^{[l]})`。
*   `n^{[l]}` 是第 `l` 层的神经元数量。

### 4.3 He初始化

由Kaiming He等人在2015年提出，主要针对ReLU及其变种（如PReLU）激活函数。

**核心思想：**
ReLU函数不是零中心化的，且会将一半的输入置为0，这改变了方差。He初始化考虑了ReLU的这一特性。

**推导结果：**
为了在使用ReLU时保持方差稳定，需要：
```
Var(W^{[l]}) = 2 / n^{[l-1]}
```

**实现方式：**
*   从高斯分布 `N(0, σ²)` 中采样，其中 `σ² = 2 / n^{[l-1]}`。
*   或从均匀分布 `U(-limit, limit)` 中采样，其中 `limit = √(6 / n^{[l-1]})`。

### 4.4 LeCun初始化

较早提出的一种初始化方法，主要针对Tanh激活函数。

**实现方式：**
*   从高斯分布 `N(0, σ²)` 中采样，其中 `σ² = 1 / n^{[l-1]}`。

## 5. 如何选择初始化方法？

*   **Sigmoid/Tanh激活函数：** 推荐使用 **Xavier** (Glorot) 初始化。
*   **ReLU及其变种（Leaky ReLU, PReLU, ELU）：** 推荐使用 **He** 初始化。
*   **Softmax输出层：** 通常使用较小的随机初始化（如Xavier）即可。
*   **现代深度学习框架：** 如TensorFlow/Keras和PyTorch，通常会根据你选择的激活函数为层自动设置默认的初始化方法（例如，`Dense`层默认使用Glorot，`Conv`层默认使用Glorot，而一些高级层可能会默认使用He）。

## 6. 其他初始化考虑

*   **偏置初始化：** 通常初始化为0是安全的。在某些情况下（如LSTM），可能会有特殊的初始化建议。
*   **预训练模型：** 如果使用预训练模型的权重进行初始化（迁移学习），则直接加载预训练的权重，无需随机初始化。
*   **批量归一化（BatchNorm）：** 如果网络中使用了BatchNorm层，初始化的影响会被部分削弱，因为BatchNorm本身会对激活值进行归一化。但良好的初始化仍然有益。

## 7. 总结

*   **参数初始化**是神经网络训练中不可忽视的重要步骤。
*   **零初始化**会导致对称性问题，使网络无法有效学习。
*   **过大或过小的随机初始化**会导致前向传播时激活值爆炸/消失，或反向传播时梯度消失/爆炸。
*   **Xavier (Glorot) 初始化**适用于Sigmoid/Tanh激活函数，旨在保持前向和反向传播时的方差稳定。
*   **He 初始化**适用于ReLU及其变种激活函数，考虑了ReLU的非线性特性。
*   选择正确的初始化方法可以显著加快训练收敛速度，提高模型性能。
*   现代深度学习框架通常会为常用层和激活函数提供合理的默认初始化策略。
